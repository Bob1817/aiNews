# Chat Page Override

本文件覆盖 `design-system/ai-news-tool/MASTER.md` 中与聊天页相关的局部规则。

## 页面定位

聊天页不是普通 IM 界面，而是“新闻编辑工作台”。

目标体验：
- 上半部分用于快速扫描新闻流
- 中部用于沉浸式阅读和创作
- 底部输入区始终保持清晰、稳定、可立即操作

## 聊天页专属规则

1. 顶部必须包含页面级定位信息。
说明：保留 hero / eyebrow / 指标卡，帮助用户理解这里是编辑台而非单纯聊天框。

2. 新闻卡片必须优先支持“快扫”。
说明：来源、时间、标题、摘要、引用动作必须在一屏内快速识别；不要把卡片做成高噪音营销风格。

3. 对话气泡必须区分“用户指令”和“AI 产出”。
说明：用户气泡可以更强烈，AI 气泡需要更像稿件面板，适合长文本阅读。

4. 空状态必须提供创作引导。
说明：不是只显示“暂无消息”，而要告诉用户如何从新闻流进入创作。

5. 输入区必须固定为主操作区。
说明：输入框、已引用状态、发送按钮需要在视觉上保持主优先级，不能被弱化成普通工具栏。

6. 加载态必须表达“编辑流程正在继续”。
说明：保留轻量动效和文案，不要使用突兀阻塞弹层。

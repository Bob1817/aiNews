# AI 助手平台化改造 - 开发任务文档

## 1. 本轮目标
在当前仓库上完成一次可运行的首版平台化改造，使项目从 `AI 新闻助手` 升级为 `AI 助手`，并具备以下最小可用能力：
- 聊天首页成为主入口。
- 支持内置与自定义工作流。
- 支持 `/名称` 与 `/+名称` 调用工作流。
- AI 能按工作流定义执行。
- 保留新闻助手作为内置工作流。

## 3. 开发步骤
### Phase A: 基础骨架
1. 增加共享工作流类型。
2. 增加后端工作流服务与工作流路由。
3. 让 `/api/ai/chat` 支持工作流命令解析。
4. 内置注册 `新闻助手` 工作流。

### Phase B: 工作流 UI
1. 新建工作流管理页面。
2. 支持工作流查看、创建、编辑、删除。
3. 对内置工作流显示只读态。

### Phase C: 聊天工作台
1. 重构聊天页布局，突出聊天主区。
2. 实现 slash command 候选面板。
3. 支持预选工作流后发送自然语言任务。
4. 右侧展示工作流列表、最近执行记录、新闻上下文。

### Phase D: 回归与收口
1. 修复类型与测试。
2. 更新文案与 README。
3. 验证普通对话、工作流调用、新闻助手、内容保存链路。

## 4. 后续迭代建议
- 为工作流增加持久化存储。
- 增加工作流导入导出。
- 增加更强的参数化输入表单。
- 增加执行日志、失败重试和权限控制。
- 让不同工作流声明自己依赖的外部配置项。

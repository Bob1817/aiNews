# 个税报表审核台设计

## 目标

将当前 `/news/file/:id` 的个税报表预览/编辑页，从通用 Excel 工作台重构为更接近正式财务系统的“报表审核台”。页面要优先服务个税申报表这种结构化模板文件，强调：

- 克制、规整、可信的专业气质
- 清晰的只读/编辑/未保存状态
- 更强的表格可读性和审核效率
- 可持续扩展到后续的校验提示、冻结列、差异标记

本次不改变底层数据读写协议，只优化页面结构、视觉层级与交互反馈。

## 设计方向

采用“专业财务工作台 + 报表审核台”方向，而不是通用在线 Excel 编辑器。

核心判断：

- 页面主任务不是自由创作，而是审核、核对、修正标准模板数据
- 操作应围绕“状态确认”和“结果提交”展开，而不是围绕编辑器工具展开
- 页面应像正式申报/财务系统，而不是像消费级文档应用

## 信息架构

页面分为三层：

1. 顶部文件抬头区
2. 左侧报表目录区
3. 右侧主审核区

### 1. 顶部文件抬头区

顶部改为正式文件抬头，不再使用 `EXCEL WORKBENCH` 这种英文产品语义。

抬头区内容：

- 返回任务结果
- 文件类型标签：`个税申报表`
- 文件名
- 更新时间
- 当前模式：`只读预览` / `编辑中`
- 编辑状态：`已保存` / `已修改未保存`

操作按钮顺序：

- `切换只读/编辑`
- `下载文件`
- `保存修改`
- `删除文件`

按钮层级规则：

- `保存修改` 为唯一主按钮
- `切换只读/编辑` 和 `下载文件` 为次按钮
- `删除文件` 为危险按钮，视觉上明确分离

## 2. 左侧报表目录区

左侧不再叫 `Sheet 列表`，改为更正式的 `报表目录`。

目录区内容：

- 报表目录标题
- 收起/展开按钮
- sheet 列表
- 当前 sheet 高亮

交互要求：

- 支持收起/展开
- 收起后保留窄栏导航能力
- 只有一个 sheet 时也保留目录，但弱化信息密度
- 当前 sheet 要有明确选中态，不依赖颜色 alone

目录区视觉规则：

- 比主表格区更轻，不抢主内容
- 收起后宽度固定为窄栏
- 展开后宽度保持稳定，避免大幅抖动

## 3. 右侧主审核区

主审核区承担全部核心任务：

- 当前 sheet 名称
- 模式说明
- 表格主体

主审核区不再强调“编辑器”概念，而强调“当前正在审核哪张申报表”。

副标题文案规则：

- 只读模式：强调当前是审核预览
- 编辑模式：强调可以修改并保存回原始文件
- 有未保存修改时：强调当前页存在待保存变更

## 表格设计规则

表格需要更接近正式财务系统。

### 表头

- 表头背景比正文更明确
- 字重高于正文
- 冻结式视觉感更强
- 标题行应有更稳定的边界和分组感

### 单元格

- 文字列左对齐
- 数值列右对齐
- 身份证号等长编号列使用更稳的数字呈现方式
- 单元格内边距统一，不再显得过松

### 状态

编辑状态需要有三层反馈：

- 当前激活单元格
- 已修改但未保存的单元格
- 只读不可编辑单元格

反馈原则：

- 克制，不使用夸张高亮
- 优先使用边框、底色、细标签表达状态
- 不影响整体财务系统的正式感

### 可读性

- 减少当前页面的大面积空白感
- 表格容器视觉上更扎实
- 列头、正文、状态提示之间层级更明确

## 视觉系统

### 色彩

采用蓝灰主色，少量琥珀色用于提醒，避免紫色、粉色、霓虹感。

建议语义：

- 主色：深蓝
- 次色：浅蓝
- 背景：冷白
- 边框：浅灰蓝
- 危险：标准红
- 提醒：低饱和琥珀

### 字体与排版

保持现有项目字体体系，不强行引入全局新字体，但页面风格参照“技术型金融/报表工作台”：

- 标题更硬朗
- 元信息更紧凑
- 数字更稳定
- 英文标签尽量去掉或弱化

### 形状与阴影

相较当前版本：

- 减少过大的圆角
- 阴影更浅更收敛
- 用边框和分区而不是大阴影建立层级

## 页面状态

需要明确覆盖以下状态：

### 1. 加载中

- 显示工作簿加载状态
- 主框架稳定，避免页面跳动

### 2. 只读预览

- 表格可浏览
- 输入框外观弱化为只读态
- 顶部显示 `只读预览`

### 3. 编辑中

- 单元格进入可编辑态
- 当前输入焦点清晰
- 主按钮为可提交状态

### 4. 已修改未保存

- 顶部出现明确但克制的脏状态提示
- 保存按钮维持主操作地位

### 5. 保存中

- 保存按钮显示进行中
- 避免误触重复提交

## 交互细节

### 收起/展开目录

- 动画时间 150ms-220ms
- 使用 transform / opacity，不做大范围布局抖动
- 收起后主表格区立即获得更多宽度

### 模式切换

- `只读预览` 与 `编辑中` 的切换要明确
- 切换文案与状态标签同步变化
- 切换后不应让用户丢失当前 sheet 位置

### 保存反馈

- 点击保存后需要明确反馈
- 若成功，脏状态清除
- 若失败，需保留当前编辑内容并显示失败信息

## 不做的内容

本次明确不包含：

- 复杂公式编辑器
- 合并单元格编辑
- 样式编辑
- 列宽拖拽
- 多文件对比视图
- 行级审计历史
- 单元格批注系统

这些能力后续如需要，可在当前审核台结构上继续扩展。

## 实现边界

本次主要改动文件：

- `src/pages/SavedFileWorkbench.tsx`

必要时允许抽出局部子组件，但不强制把页面拆成大量新文件。优先保证：

- 结构层级清楚
- 样式 token 可维护
- 交互状态可读

## 验收标准

完成后应满足：

- 页面一眼看上去像正式报表审核系统，而不是通用文档编辑器
- 顶部能清楚表达文件身份、模式和保存状态
- 左侧目录收起后，表格区明显获得更多空间
- 只读和编辑模式差异清晰
- 表格更易读，尤其是编号列、金额列、字段标题
- 用户在编辑时能明确知道哪些内容改过但未保存

# Excel 文件工作台设计

## 背景

当前 `任务结果 -> 文件` 已经能展示由 `/个税报表` 工作流生成的 Excel 文件记录，但这些记录仍然只具备基础下载和删除能力。用户新增需求是让这类 Excel 文件支持在线预览、在线编辑单元格、下载和删除，且编辑对象必须是 Excel 文件本身，而不是对话内容或任务摘要。

这意味着系统需要把 `xlsx` 文件视为一类可管理资产，而不是仅把它作为“可下载附件”挂在任务结果卡片上。

## 目标

- 在 `任务结果 -> 文件` 中为 `xlsx` 文件提供专属入口
- 支持在线预览 Excel 文件内容
- 支持按 sheet 查看和切换
- 支持编辑单元格值并保存回原始 xlsx 文件
- 保留文件下载和删除能力
- 删除时同步删除任务结果记录和磁盘文件

## 非目标

- 不支持复杂 Excel 样式编辑
- 不支持合并单元格、批量公式编辑、条件格式、图表编辑
- 不处理对话内容编辑
- 不把所有文件类型都升级为工作台，仅优先支持 `xlsx`

## 方案选择

### 方案 A：独立 Excel 详情页工作台

从 `任务结果` 列表点击进入专属详情页，例如 `/news/file/:id`。页面结构为顶部文件工具栏、左侧 sheet 列表、右侧表格预览/编辑区。

优点：

- 空间充足，适合长表格和多 sheet
- 预览与编辑状态切换清楚
- 后续扩展冻结表头、更多元信息更容易

缺点：

- 比弹窗或抽屉多一次页面跳转

### 方案 B：列表页右侧抽屉

在 `任务结果` 页内保留列表，点击文件后从右侧拉出预览/编辑抽屉。

优点：

- 跳转少

缺点：

- 多 sheet 和大表格编辑空间不足
- 文件动作和列表筛选容易互相打扰

### 结论

采用方案 A。Excel 是重内容文件，独立详情页更适合预览、编辑和保存回写。

## 信息架构

### 列表页

在 `任务结果 -> 文件` 列表中，`xlsx` 文件卡片提供以下动作：

- `预览`
- `编辑`
- `下载`
- `删除`

其中：

- `预览` 跳转到文件详情页，只读模式
- `编辑` 跳转到文件详情页，直接进入编辑模式
- `下载` 复用现有文件下载接口
- `删除` 保持现有确认交互，但删除时需同步移除磁盘文件

### Excel 文件详情页

页面由三部分构成：

1. 顶部工具栏
   - 返回任务结果
   - 文件标题
   - 更新时间 / 文件类型
   - 编辑模式切换
   - 保存
   - 下载
   - 删除

2. 左侧 sheet 导航
   - 展示工作簿内所有 sheet
   - 当前 sheet 高亮
   - 点击切换当前 sheet

3. 右侧表格工作区
   - 默认以表格形式渲染当前 sheet
   - 只读模式不可修改
   - 编辑模式允许单元格值变更
   - 支持纵向滚动和横向滚动

## 交互设计

### 预览模式

- 进入 `/news/file/:id?mode=preview`
- 单元格不可编辑
- 保留下载、删除、切换 sheet
- 右上角有 `进入编辑` 按钮

### 编辑模式

- 进入 `/news/file/:id?mode=edit`
- 单元格可点击编辑
- 修改后页面出现“未保存更改”状态
- 点击 `保存` 后调用后端回写 Excel
- 保存成功后更新时间刷新

### 删除

- 在详情页删除时，沿用确认弹层
- 删除成功后返回任务结果列表
- 同时删除 `SavedNews` 记录和磁盘文件

## 后端设计

为 `SavedNews` 中的 `xlsx` 文件增加专属文件接口：

1. `GET /api/news/saved/:id/workbook`
   - 校验记录存在且为 `outputType=file && fileFormat=xlsx`
   - 读取本地 xlsx 文件
   - 返回：
     - 文件基础信息
     - sheet 名称列表
     - 当前工作簿各 sheet 的二维单元格数据

2. `PUT /api/news/saved/:id/workbook`
   - 接收修改后的 sheet 单元格值
   - 回写原始 xlsx 文件
   - 更新对应 `SavedNews.updatedAt`
   - 返回最新记录和工作簿摘要

3. `DELETE /api/news/saved/:id`
   - 若为文件记录且磁盘文件存在，同时删除文件
   - 再删除 `SavedNews` 记录

### 数据返回结构

建议新增工作簿视图模型：

- `sheetNames: string[]`
- `sheets: Array<{ name: string; rows: string[][] }>`
- `activeSheet?: string`

第一版只返回纯单元格文本值，不返回样式、公式、批注等复杂结构。

## 前端设计

### 路由

新增页面：

- `/news/file/:id`

默认读取 `mode=preview | edit` 查询参数。

### 组件拆分

建议新增：

- `src/pages/FileWorkbench.tsx`
- `src/components/excel/WorkbookSheetList.tsx`
- `src/components/excel/WorkbookGrid.tsx`
- `src/lib/api/workbook.ts`

职责如下：

- `FileWorkbench`：页面状态、加载、保存、删除、模式切换
- `WorkbookSheetList`：sheet 导航
- `WorkbookGrid`：当前 sheet 表格渲染与单元格编辑
- `workbook.ts`：请求读取/保存工作簿 API

### 状态管理

页面本地状态足够，不必引入 store：

- `workbook`
- `activeSheet`
- `isEditing`
- `draftCells`
- `isSaving`
- `hasUnsavedChanges`

## 表格渲染策略

第一版采用 HTML table 渲染：

- 第一行/列不强行解释为表头，仅忠实展示单元格
- 空值显示为空字符串
- 单元格宽度采用最小宽度并允许横向滚动
- 编辑时用 input 覆盖当前单元格

不引入重量级 spreadsheet 组件库，先用现有 React + Tailwind 实现最小闭环。

## 删除与下载一致性

- 下载继续使用现有 `downloadSavedNewsFile`
- 删除时如果 `outputType=file` 且存在 `filePath`，服务端先尝试删除文件，再删除记录
- 若文件已不存在，但记录存在，仍允许删除记录

## 错误处理

- 文件不存在：详情页显示“文件已丢失”，仍允许删除记录
- 读取失败：展示错误提示并允许返回列表
- 保存失败：保留未保存修改，不清空本地编辑状态
- 非 xlsx 文件访问工作簿页：直接提示“不支持此文件类型”

## 测试策略

### 后端

- 读取 workbook 成功
- 保存 workbook 成功并更新文件
- 删除文件记录时同步删除磁盘文件
- 文件丢失时仍允许删除记录
- 非 xlsx 文件访问 workbook 接口时报错

### 前端

- `任务结果` 中 xlsx 文件出现 `预览/编辑/下载/删除`
- 文件工作台能正确加载 sheet 列表和表格内容
- 只读模式不可编辑
- 编辑模式修改单元格后可保存
- 删除成功后跳回列表

## 风险与后续演进

### 当前风险

- HTML table 在超大表格上性能一般
- 不处理复杂 Excel 语义时，公式和样式可能只能以结果值形式展示

### 当前缓解

- 第一版只针对个税模板这类结构相对稳定的文件
- 若后续出现大表格需求，再考虑虚拟滚动或专业表格组件

## 后续可扩展项

- 支持冻结首行/首列
- 支持单元格批量粘贴
- 支持只保存修改过的 diff
- 支持更多文件格式，如 `csv`

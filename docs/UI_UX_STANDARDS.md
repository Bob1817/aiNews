# UI / UX 开发规范

本项目后续所有前端页面、组件和交互开发，必须遵循以下规范。

设计系统主文件：
- [design-system/ai-news-tool/MASTER.md](/Users/shibo/Documents/trae_projects/AINews/design-system/ai-news-tool/MASTER.md)

如果未来某个页面需要偏离全局规则，必须在 `design-system/ai-news-tool/pages/<page>.md` 中声明覆盖项；没有页面级覆盖时，一律以 `MASTER.md` 为准。

## 强制规则

1. 视觉方向必须保持“编辑台 / 新闻运营台”风格。
说明：深色主背景、强层级对比、编辑感标题排版、克制但明确的渐变点缀；禁止回退到普通白底 SaaS 模板风格。

2. 所有颜色必须使用语义 token，不允许在组件里随意写裸十六进制。
说明：优先使用 `src/index.css` 和 `tailwind.config.js` 中已定义的 token、主题色和字体体系。

3. 标题统一使用 `Newsreader`，正文与交互文字统一使用 `Roboto`。
说明：标题负责建立“新闻/编辑”气质，正文负责高密度信息可读性；不要混入新的展示字体。

4. 页面结构必须移动优先，并同时通过 375、768、1024、1440 四档检查。
说明：禁止出现水平滚动、内容被固定元素遮挡、超小点击区域等问题。

5. 所有交互元素必须保留明确的 focus、hover、disabled 状态。
说明：不能删除可访问性焦点样式；图标按钮必须有 `aria-label`。

6. 动效只用于表达层级和状态变化，时长控制在 150ms 到 300ms。
说明：禁止装饰性大动画、阻塞输入的动画、会造成布局抖动的 hover 效果。

7. 组件层级必须统一。
说明：主容器使用 `surface-panel` 或 `surface-panel-soft`；卡片、弹层、消息气泡、状态块不要混用多套阴影和圆角语言。

8. 表单与反馈必须就近呈现。
说明：错误信息贴近字段或操作区域；加载、成功、失败状态需要有清晰反馈； destructive 操作必须具备危险态样式和确认路径。

9. 导航结构必须稳定。
说明：一级导航、设置区、危险操作必须清晰分区；不要在不同页面随意变更导航位置和语义。

10. 新页面开发前，必须先读设计系统文件。
说明：开发流程固定为：
- 先读 `design-system/ai-news-tool/MASTER.md`
- 若存在页面级文件，再读 `design-system/ai-news-tool/pages/<page>.md`
- 然后再开始写 UI 代码

## 禁止事项

- 禁止使用 emoji 作为结构性图标
- 禁止使用没有标签的纯图标主要操作
- 禁止用颜色作为唯一状态表达
- 禁止在组件中新增一套与当前主题冲突的卡片/按钮风格
- 禁止无理由使用纯白大面积页面背景
- 禁止新增不受控的动画库效果或高饱和渐变污染主界面

## 落地建议

- 优先复用现有样式基元：`surface-panel`、`surface-panel-soft`、`eyebrow`、`focus-ring`
- 页面文案层级建议：
  - Section 标签：全大写、较宽字距、小号字
  - 主标题：`Newsreader`，高对比、大字号
  - 辅助文案：`Roboto`，行高 1.6 左右，使用 muted 色
- 需要新增公共视觉基元时，应先更新全局 token，再在组件中消费，不要直接局部硬编码

## 评审要求

以后所有 UI PR / 提交在合入前，至少自查以下问题：

- 是否符合 `design-system/ai-news-tool/MASTER.md`
- 是否保留键盘焦点和无障碍标签
- 是否在深色背景下仍有足够对比度
- 是否在 375px 宽度下无布局破坏
- 是否复用了项目已有主题 token 和公共基元

# 部署检查清单

## 上线前准备

### 1. 环境配置
- [ ] 创建 `.env.production` 文件并填写所有必要配置
- [ ] 更新 `JWT_SECRET` 为至少 32 个字符的强随机密钥
- [ ] 配置正确的 `CORS_ORIGINS` 域名白名单
- [ ] 配置所有第三方 API 密钥（NewsAPI、OpenAI、Supabase等）
- [ ] 配置数据库连接字符串（如使用）
- [ ] 设置 `NODE_ENV=production`

### 2. 安全检查
- [ ] 确保所有敏感信息都在环境变量中，不在代码中硬编码
- [ ] 检查 `.gitignore` 文件确保 `.env` 等敏感文件不会被提交
- [ ] 验证 Helmet 安全头配置正确
- [ ] 验证速率限制配置生效
- [ ] 验证 SQL 注入和 XSS 防护中间件正常工作

### 3. 构建优化
- [ ] 运行 `npm run build` 构建前端
- [ ] 运行 `npm run build:api` 构建后端
- [ ] 检查构建产物大小，确保没有过大的 chunk
- [ ] 验证 sourcemap 只在开发环境生成
- [ ] 验证生产构建已移除 console.log 和 debugger

### 4. 测试验证
- [ ] 运行单元测试：`npm run test`
- [ ] 运行类型检查：`npm run typecheck`
- [ ] 运行 ESLint 检查：`npm run lint`
- [ ] 手动测试核心功能流程
- [ ] 测试错误边界和错误处理

### 5. 性能优化
- [ ] 验证代码分割（Code Splitting）配置正确
- [ ] 验证第三方库合理分包
- [ ] 检查静态资源压缩配置
- [ ] 配置 CDN（如需要）

## 部署步骤

### 6. 服务器准备
- [ ] 配置 Node.js 环境（推荐 LTS 版本）
- [ ] 配置反向代理（Nginx/Apache）
- [ ] 配置 SSL 证书（HTTPS）
- [ ] 配置进程管理器（PM2/systemd）
- [ ] 配置日志轮转

### 7. 部署应用
- [ ] 上传构建产物到服务器
- [ ] 安装生产依赖：`npm ci --only=production`
- [ ] 配置环境变量
- [ ] 启动应用服务
- [ ] 验证健康检查端点：`/api/health`

### 8. 监控和维护
- [ ] 配置应用监控（如需要）
- [ ] 配置错误告警
- [ ] 配置日志收集
- [ ] 建立定期备份策略
- [ ] 文档化部署流程

## 回滚计划
- [ ] 保留上一个稳定版本的备份
- [ ] 准备快速回滚脚本
- [ ] 验证数据库回滚策略（如需要）
